🧬 Phân Tích Hình Thái Nhiễm Sắc Thể (NST)

Hệ thống AI tự động phân vùng và tách hai nhiễm sắc thể chồng nhau trong ảnh karyotype. Nhận diện 3 trạng thái: NST A (tách), NST B (tách), và vùng C (chồng lấp). Sử dụng kiến trúc Hybrid Attention U-Net theo mô hình Teacher-Student.

🎯 Segmentation 🧠 Teacher-Student Distillation 👁️ Attention U-Net 🔬 Synthetic Data Generation 🎨 OpenCV Inpainting 📐 256×256 Input

def get_border_background(gray):
    """
    [HÀM LÕI] Phát hiện nền bằng phân tích vùng biên.
    
    Ý tưởng then chốt: Nền thật = vùng SÁNG + TIẾP XÚC với biên ảnh.
    Vùng trắng bên trong NST → không chạm biên → không bị xóa.
    
    Thuật toán:
        1. Lấy pixel ở 4 cạnh ảnh → tính median → màu nền tham chiếu.
        2. Tìm tất cả pixel sáng hơn ngưỡng (candidate background).
        3. Dùng connectedComponents → giữ chỉ những component
           có pixel CHẠM biên ảnh = nền thật.
    """
    h, w = gray.shape

    # Lấy pixel biên để ước lượng màu nền
    border_pixels = np.concatenate([
        gray[0, :], gray[h-1, :],   # cạnh trên + dưới
        gray[:, 0], gray[:, w-1]    # cạnh trái + phải
    ])
    bg_ref = np.median(border_pixels)
    threshold = max(210, bg_ref - BG_MARGIN)

    # Pixel sáng hơn ngưỡng = candidate background
    candidate_bg = gray >= threshold

    # Tìm component nào chạm biên = nền thật
    num_labels, labels, stats, _ = cv2.connectedComponentsWithStats(
        candidate_bg.astype(np.uint8), connectivity=8
    )
    border_labels = set(np.unique(np.concatenate([
        labels[0, :], labels[h-1, :],
        labels[:, 0], labels[:, w-1]
    ])))
    border_labels.discard(0)  # bỏ label 0 (không phải candidate)

    background = np.isin(labels, list(border_labels)) & candidate_bg
    return background, threshold, bg_ref


def extract_object_rgba(img_path):
    """
    [HÀM CHÍNH] Tách NST khỏi nền → RGBA với nền trong suốt.
    
    Quy trình:
        1. Đọc ảnh → grayscale.
        2. get_border_background() → xác định nền.
        3. Morphology open + close → làm sạch mask.
        4. keep_largest_component() → bỏ noise.
        5. Erode 1px → giảm halo trắng ở viền.
        6. Crop sát + padding nhỏ.
        7. Tạo RGBA: pixel NST giữ nguyên, pixel nền → alpha=0.
    """
    img = Image.open(img_path).convert("RGB")
    arr = np.array(img)
    gray = np.array(img.convert("L"))

    background, threshold, bg_ref = get_border_background(gray)
    object_mask = ~background   # NST = KHÔNG phải nền

    # Làm sạch mask
    kernel = np.ones((KERNEL_SIZE, KERNEL_SIZE), np.uint8)
    object_mask = cv2.morphologyEx(object_mask.astype(np.uint8)*255,
                                   cv2.MORPH_OPEN, kernel)
    object_mask = cv2.morphologyEx(object_mask, cv2.MORPH_CLOSE, kernel) > 0
    object_mask = keep_largest_component(object_mask)

    # Co mask nhẹ để giảm viền trắng giả
    if MASK_SHRINK_ITER > 0:
        object_mask = cv2.erode(object_mask.astype(np.uint8)*255,
                               kernel, iterations=MASK_SHRINK_ITER) > 0

    # Crop sát object
    ys, xs = np.where(object_mask)
    x1, x2 = xs.min()-CROP_PADDING, xs.max()+CROP_PADDING
    y1, y2 = ys.min()-CROP_PADDING, ys.max()+CROP_PADDING

def make_realistic_overlap_canvas(obj_A, mask_A, obj_B, mask_B):
    """
    [HÀM LÕI] Tạo 1 cặp ảnh NST chồng nhau có kiểm soát.
    
    Chiến lược:
        - A xoay ~90° ± 8° → nằm ngang
        - B xoay ~0° ± 8° → đứng
        - Tâm 2 NST đặt gần trung tâm canvas, lệch nhau ±10-18px
        
    Từ chối mẫu nếu:
        - Overlap < 120px (chồng quá ít, không đại diện)
        - Overlap > 55% diện tích NST nhỏ hơn (chồng quá nhiều)
    
    C = A ∩ B  (giao nhị phân pixel-by-pixel)
    """
    # 1. Resize ngẫu nhiên
    target_A = random.randint(TARGET_LONG_SIDE_MIN, TARGET_LONG_SIDE_MAX)
    target_B = random.randint(TARGET_LONG_SIDE_MIN, TARGET_LONG_SIDE_MAX)
    obj_A, mask_A = resize_keep_ratio(obj_A, mask_A, target_A)
    obj_B, mask_B = resize_keep_ratio(obj_B, mask_B, target_B)

    # 2. Xoay: A nằm ngang, B đứng
    angle_A = 90 + random.uniform(-8, 8)
    angle_B = random.uniform(-8, 8)
    obj_A, mask_A = rotate_pair(obj_A, mask_A, angle_A)
    obj_B, mask_B = rotate_pair(obj_B, mask_B, angle_B)

    # 3. Canvas trắng, tâm gần nhau
    canvas = Image.new("RGBA", (CANVAS_SIZE, CANVAS_SIZE), (255,255,255,255))
    center_x = CANVAS_SIZE//2 + random.randint(-15, 15)
    center_y = CANVAS_SIZE//2 + random.randint(-15, 15)
    A_cx = center_x + random.randint(-18, 18)
    B_cy = center_y + random.randint(-18, 18)

    # 4. Dán ngẫu nhiên thứ tự (A trước B hoặc B trước A)
    if random.random() < 0.5:
        canvas, mask_A_canvas = paste_object(canvas, mask_A_canvas, obj_A, mask_A, A_cx, A_cy)
        canvas, mask_B_canvas = paste_object(canvas, mask_B_canvas, obj_B, mask_B, B_cx, B_cy)
    else:
        canvas, mask_B_canvas = paste_object(canvas, mask_B_canvas, obj_B, mask_B, B_cx, B_cy)
        canvas, mask_A_canvas = paste_object(canvas, mask_A_canvas, obj_A, mask_A, A_cx, A_cy)

    # 5. C = giao nhị phân pixel
    A_arr = np.array(mask_A_canvas) > 0
    B_arr = np.array(mask_B_canvas) > 0
    C_arr = A_arr & B_arr   # ← C chính xác 100%!

    # 6. Kiểm soát chất lượng overlap
    overlap_pixels = int(C_arr.sum())
    overlap_ratio = overlap_pixels / min(A_arr.sum(), B_arr.sum())
    if overlap_pixels < MIN_OVERLAP_PIXELS: return None
    if overlap_ratio > MAX_OVERLAP_RATIO:   return None

    return final_image, mask_A_canvas, mask_B_canvas, mask_C_canvas

4v1_preprocess_to_256.py — Chuẩn Hóa Ảnh
Tại sao dùng PADDING thay vì STRETCH? Stretch làm méo hình thái NST. Hình dạng NST là đặc trưng quan trọng trong karyotype, không được biến dạng.
Python 4v1 — resize_with_padding + binarize_mask 
def resize_with_padding(img, target_size=256, is_mask=False):
    """
    Resize giữ tỉ lệ → paste vào tâm canvas target×target.
    
    Ảnh (is_mask=False):  Resize BILINEAR + padding TRẮNG (255)
    Mask (is_mask=True):  Resize NEAREST + padding ĐEN (0)
    
    Lý do padding khác nhau:
        - Ảnh: nền trắng khớp với nền tự nhiên của NST karyotype.
        - Mask: vùng padding không có NST → phải là 0 (nền).
    """
    w, h = img.size
    scale = min(target_size/w, target_size/h)  # scale nhỏ nhất để vừa
    new_w, new_h = int(round(w*scale)), int(round(h*scale))

    resample = Image.NEAREST if is_mask else Image.BILINEAR
    fill = 0 if is_mask else 255
    img = img.resize((new_w, new_h), resample)
    canvas = Image.new("L", (target_size, target_size), fill)

    # Paste vào TÂM canvas
    paste_x = (target_size - new_w) // 2
    paste_y = (target_size - new_h) // 2
    canvas.paste(img, (paste_x, paste_y))
    return canvas


def binarize_mask(mask_img):
    """
    Đảm bảo mask chỉ có 2 giá trị: 0 hoặc 255.
    Gọi CẢ TRƯỚC VÀ SAU resize vì BILINEAR có thể tạo giá trị trung gian.
    """
    arr = np.array(mask_img)
    arr = (arr > 127).astype(np.uint8) * 255
    return Image.fromarray(arr, mode="L")

📡 Output Channels (4 kênh dự đoán)

A
Vùng NST thứ nhất
Mask vùng thuộc riêng NST A
Threshold: 0.50 · Weight: 1.25
B
Vùng NST thứ hai
Mask vùng thuộc riêng NST B
Threshold: 0.50 · Weight: 1.25
C
Vùng Chồng Lấp (Overlap)
A ∩ B · Vùng hai NST đè lên nhau
Threshold: 0.40 · Weight: 3.75 ⭐
E
Edge (Đường Biên)
Auxiliary channel · Không xuất ra kết quả cuối
Tự động sinh từ A∪B∪C · Weight: 1.00
Tại sao C có weight 3.75? Vùng C (overlap) chiếm diện tích nhỏ. Nếu không có trọng số cao, model sẽ "bỏ qua" C vì contribution vào loss nhỏ. 3.75 = 3 × 1.25 → C quan trọng bằng tổng A + B + 50%.

CURRENT_HARD_LOSS_WEIGHT  = tf.Variable(0.65, trainable=False, dtype=tf.float32)
CURRENT_DISTILL_LOSS_WEIGHT = tf.Variable(0.35, trainable=False, dtype=tf.float32)

def student_distill_loss(y_true: tf.Tensor, y_pred: tf.Tensor) -> tf.Tensor:
    """
    Loss function cho Student trong Teacher-Student Distillation.
    
    y_true có 8 channels = concat(hard_ABCE, soft_ABCE):
        - channels 0..3: nhãn cứng từ ground truth (0/1)
        - channels 4..7: soft output của Teacher (xác suất 0..1)
    
    Loss = 0.65 × hybrid_loss(hard, pred)      ← học từ ground truth
         + 0.35 × MSE(soft, pred) × weights    ← học từ Teacher
    
    Tại sao 65/35?
        100% hard = mất lợi ích distillation.
        100% soft = mất độ chính xác với ground truth.
        65/35 = cân bằng tốt nhất theo thực nghiệm.
    """
    hard = y_true[..., :NUM_OUTPUT_CHANNELS]           # channels 0-3
    soft = y_true[..., NUM_OUTPUT_CHANNELS:NUM_OUTPUT_CHANNELS*2]  # channels 4-7

    hard_loss = hybrid_loss(hard, y_pred)

    # MSE giữa Teacher output và Student output, weighted theo kênh
    mse = tf.square(tf.cast(soft, tf.float32) - tf.cast(y_pred, tf.float32))
    distill = tf.reduce_mean(mse * CHANNEL_WEIGHTS)

    return (CURRENT_HARD_LOSS_WEIGHT * hard_loss
          + CURRENT_DISTILL_LOSS_WEIGHT * distill)


def make_boundary_channel(mask_abc: tf.Tensor, k: int = 3) -> tf.Tensor:
    """
    Sinh edge channel từ union(A,B,C) bằng morphology trên TensorFlow.
    Edge = Dilation(fg) - Erosion(fg)  ← đường viền mask
    
    Không cần label edge thủ công — tự động từ mask A/B/C!
    """
    fg = tf.reduce_max(mask_abc, axis=-1, keepdims=True)   # union A∪B∪C
    x = tf.expand_dims(fg, axis=0)
    dil = tf.nn.max_pool2d(x, ksize=k, strides=1, padding="SAME")
    ero = 1.0 - tf.nn.max_pool2d(1.0-x, ksize=k, strides=1, padding="SAME")
    edge = tf.clip_by_value(dil - ero, 0.0, 1.0)
    return tf.squeeze(edge, axis=0)

def inpaint_instance(gray, instance_mask, overlap_mask, radius=3):
    """
    [HÀM LÕI] Fill vùng bị che khuất (C) bằng TELEA inpainting.
    
    Vấn đề: NST A bị NST B che ở vùng C → texture vùng đó là của B.
    Giải pháp: 
        1. Mask vùng ngoài A thành TRẮNG → loại texture B
        2. Dilate nhẹ vùng fill → tránh viền còn sót
        3. cv2.INPAINT_TELEA → đoán texture từ vùng A visible
    
    Tại sao TELEA thay vì Navier-Stokes?
        NST là ảnh y tế đen trắng đơn giản. TELEA (Fast Marching)
        cho kết quả mượt hơn cho texture đơn giản.
    """
    gray_u8 = gray.astype(np.uint8)
    work = gray_u8.copy()
    work[~instance_mask] = 255  # ← TRICK: xóa texture NST kia

    fill_region = (instance_mask & overlap_mask).astype(np.uint8) * 255
    if fill_region.sum() == 0: return work

    # Dilate nhẹ để không còn artifact ở biên
    kernel = np.ones((3,3), np.uint8)
    fill_region = cv2.dilate(fill_region, kernel, iterations=1)
    fill_region[~instance_mask] = 0

    return cv2.inpaint(work, fill_region, inpaintRadius=radius,
                      flags=cv2.INPAINT_TELEA)


def save_separated_ab(original_gray, stem, mask_A, mask_B, mask_C,
                      out_dir, transparent=True):
    """
    [OUTPUT CHÍNH] Tạo sản phẩm cuối cùng: 2 file NST đã tách.
    
    Tại sao OR với C?
        C là vùng overlap → thuộc về CẢ A lẫn B (amodal segmentation).
        Giữ full hình thái từng NST → phân loại karyotype chính xác.
    """
    gray = np.array(original_gray.convert("L"))

    # A_full = A ∪ C, B_full = B ∪ C (giữ full hình thái)
    A_full = clean_mask(mask_A | mask_C, keep_components=1)
    B_full = clean_mask(mask_B | mask_C, keep_components=1)
    C_clean = clean_mask(mask_C, keep_components=1)

    # Inpaint vùng C cho từng NST
    img_A = rgba_from_instance(gray, A_full, C_clean, transparent=transparent)
    img_B = rgba_from_instance(gray, B_full, C_clean, transparent=transparent)

    # Lưu 4 file output
    img_A.save(out_dir / f"{stem}_A.png")
    img_B.save(out_dir / f"{stem}_B.png")
    save_binary_mask(A_full, out_dir / f"{stem}_A_mask.png")
    save_binary_mask(B_full, out_dir / f"{stem}_B_mask.png")

📏IoU Score

IoU = |A∩B| / |A∪B|. Metric phụ, khắt khe hơn Dice. Thường thấp hơn Dice 10-20%.

🎯Pixel Accuracy

Tỉ lệ pixel phân loại đúng. Có thể cao giả khi nền chiếm nhiều → Dice thực tế hơn.

Nhật Ký Quyết Định Thiết Kế

Mỗi quyết định đều có lý do kỹ thuật rõ ràng. Đọc trước khi thay đổi bất kỳ tham số nào.

DD-01
Dùng Synthetic Data thay vì label thủ công
Ảnh NST thật (overlap_raw) KHÔNG có ground-truth mask. Label thủ công cực kỳ tốn thời gian và chủ quan. Synthetic cho mask chính xác 100% và có thể sinh không giới hạn.
→ 3v1_generate_synthetic_masks.py
DD-02
Padding thay vì Stretch khi resize về 256
Stretch làm méo hình thái NST. Trong karyotype, hình dạng NST (dài, ngắn, cong) là đặc trưng quan trọng để phân loại. Padding trắng khớp với nền tự nhiên.
→ 4v1_preprocess_to_256.py, 7v1_predict_real_overlap.py
DD-03
Channel C weight = 3.75 (gấp 3× A và B)
Vùng C chiếm diện tích nhỏ hơn A và B. Không có trọng số cao, model sẽ học cách "bỏ qua" C vì ít ảnh hưởng đến loss tổng thể. 3.75 = 3 × 1.25.
→ 6v1_unet_model.py: CHANNEL_WEIGHTS
DD-04
Threshold C = 0.40 (thấp hơn A/B = 0.50)
Vùng C nằm ở vùng chồng lấp phức tạp, model thường dự đoán với confidence thấp hơn A/B. Threshold thấp hơn giúp recall C tốt hơn.
→ 6v1_unet_model.py, 7v1, 8v1: ABC_METRIC_THRESHOLDS
DD-05
Tắt XLA auto-jit (TF_XLA_FLAGS=--tf_xla_auto_jit=0)
Epoch đầu trên Colab T4 bị treo 10-30 phút do XLA/cuDNN autotune tìm thuật toán tối ưu cho shape đầu tiên. Tắt đi để bắt đầu train ngay.
→ 6v1_unet_model.py: os.environ
DD-06
Fixed batch size chia hết cả train và val
Batch shape thay đổi giữa các epoch (batch lẻ cuối dataset) → XLA/cuDNN phải compile lại kernel → chậm đáng kể. choose_fixed_batch_size() tự động tìm batch tối ưu.
→ 6v1_unet_model.py: choose_fixed_batch_size()
DD-07
Teacher 48 filters, Student 24 filters
Teacher cần capacity lớn để học tốt. Student nhẹ hơn để inference nhanh khi deploy. Student "chắt lọc" kiến thức từ Teacher thay vì phải tự học từ đầu.
→ 6v1_unet_model.py: TrainConfig
DD-08
Distillation: 65% hard label + 35% soft label
100% hard = bỏ lợi ích distillation. 100% soft = mất độ chính xác với ground truth. 65/35 cân bằng tốt nhất theo thực nghiệm trên dataset NST.
→ 6v1_unet_model.py: student_distill_loss()
DD-09
Inpainting bằng TELEA thay vì Navier-Stokes
NST là ảnh y tế đen trắng với texture đơn giản. TELEA (Fast Marching Method, Telea 2004) cho kết quả mượt và ít artifact hơn Navier-Stokes cho loại texture này.
→ 7v1_predict_real_overlap.py: inpaint_instance()
DD-10
Augmentation chỉ dùng Flip, KHÔNG crop/rotate
Crop ngẫu nhiên có thể cắt mất phần NST quan trọng. Rotate mạnh làm lệch hướng NST (A nằm ngang / B đứng là đặc trưng quan trọng). Flip H/V an toàn.
→ 6v1_unet_model.py: augment_sample()
